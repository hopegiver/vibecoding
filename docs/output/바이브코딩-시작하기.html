WHAT IS IT

1바이브코딩이란?

한 문장으로 말하면 이렇습니다.

바이브코딩(Vibe Coding)은 AI에게 말로 부탁해서 웹사이트나 간단한 프로그램을 만드는 새로운 방식입니다.

"이런 걸 만들고 싶어"라고 한국어로 이야기하면, AI가 필요한 코드를 대신 써 줍니다. 마음에 안 들면 다시 "이렇게 바꿔줘"라고 말하면 됩니다.

요리에 비유하면

AI
최신 자동 조리기

버튼 하나로 요리를 뚝딱 만들어 줍니다.

나의 역할
재료·레시피 이해

뭘 넣을지, 맛이 이상하면 뭘 고칠지 알아야 합니다.

결과
원하는 요리 완성

조리기에 정확히 설명할수록 결과가 좋아집니다.

아무리 좋은 자동 조리기가 있어도, 원하는 요리를 설명할 줄은 알아야겠죠. 바이브코딩도 똑같습니다.

기존 방식 vs 바이브코딩

＝기존 코딩

  • 문법을 외우고 모든 코드를 직접 작성
  • 에러가 나면 검색하며 해결책 찾기
  • 배우는 데 시간이 오래 걸림
  • 개발자가 아니면 시작하기 어려움

＋바이브코딩

  • AI에게 "이런 걸 만들고 싶어"라고 말하면 끝
  • 에러가 나면 AI가 바로 해결 방법 제시
  • 몇 시간이면 실제 웹사이트 완성 가능
  • 비개발자도 오늘 바로 시작 가능

WHY BASICS MATTER

3왜 AI 시대에도 기본을 알면 좋을까

"AI가 다 해준다는데, 왜 조금이라도 알아야 하나요?" 자주 나오는 질문입니다.

기본을 조금만 알면, AI를 10배 더 잘 씁니다.
전부 배우자는 게 아닙니다. AI에게 정확히 부탁하고, 결과를 이해할 만큼만 알면 됩니다.
1
AI에게 정확하게 부탁할 수 있어요

"예쁘게 만들어줘"보다 "제목은 크게, 버튼은 파란색으로"처럼 구체적으로 말하면 원하는 결과가 나옵니다.

2
작은 부분은 직접 고칠 수 있어요

색만 바꾸고 싶은데 AI가 엉뚱한 곳을 건드릴 때, 기본을 알면 5초 만에 직접 고칩니다.

3
문제가 생겨도 당황하지 않아요

에러 메시지의 뜻을 대략 이해하면, AI에게 "여기가 문제 같아"라고 정확히 다시 물어볼 수 있습니다.

4
조금씩 발전시킬 수 있어요

매번 처음부터 다시 만들 필요 없이, 필요한 부분만 손보며 점점 완성도를 높입니다.

필요한 공부는 딱 하루 (총 3시간)

배우는 것무엇인가요걸리는 시간
HTML웹페이지의 뼈대제목·글·이미지의 자리30분
CSS디자인(색·크기·배치)겉모습을 꾸미는 것1시간
JavaScript동작·기능버튼을 누르면 일어나는 일1시간
JSON데이터 정리 형식목록·정보를 담는 상자30분
합계평생 AI를 잘 쓰기 위한 최소 지식3시간

이 문서는 이 중 "HTML로 첫 페이지 만들기"까지 안내합니다. 나머지는 만들면서 자연스럽게 익히게 됩니다.

STEP BY STEP

7따라 하기: 처음부터 끝까지

읽기만 하면 남는 게 없습니다. 지금부터 화면에 보이는 버튼·메뉴 이름 그대로 5단계를 짚어 갑니다. 빈 폴더에서 시작해 전 세계에 공개된 내 사이트가 열리는 데까지, 순서대로 그대로 따라오면 됩니다.

외울 것은 없습니다. 이 페이지를 보며 버튼을 하나씩 누르기만 하면 됩니다.
STEP 1~3 = 내 컴퓨터에서 페이지 만들기 · STEP 4~5 = 인터넷에 공개하기.

* 이 가이드의 단축키는 Windows 기준입니다. Mac 사용자는 Ctrl 대신 Cmd(⌘)를 누르세요.

STEP 1 VS Code에 Claude Code 연결하기
  1. VS Code를 실행합니다.
  2. 왼쪽 사이드바에서 확장(Extensions) 아이콘을 클릭합니다. (단축키 Ctrl + Shift + X)
  3. 검색창에 Claude Code 를 입력하고, 발행자가 Anthropic인 것을 확인한 뒤 Install 을 클릭합니다.
  4. 설치 후 VS Code가 재시작을 권하면 그대로 따릅니다.
  5. 왼쪽 활동 표시줄(Activity Bar)의 Spark 아이콘(항상 보임), 또는 오른쪽 아래 상태 표시줄의 ✱ Claude Code 를 클릭하면 패널이 열립니다. (파일을 연 뒤에는 편집기 오른쪽 위에도 아이콘이 생깁니다.) 여기서 Sign in 을 클릭합니다.
  6. 자동으로 열린 브라우저에서 claude.com에 로그인합니다(Claude Pro 구독 계정, 구글 계정으로도 로그인 가능). 승인하면 VS Code로 자동 복귀 — 연결 끝.
확인 포인트 · 발행자가 Anthropic이 맞는지 꼭 보세요. 이름만 비슷한 다른 확장이 있을 수 있습니다.
STEP 2 내 작업 폴더 만들고 열기
  1. 바탕화면에서 마우스 우클릭 → 새로 만들기 → 폴더, 이름을 my-website 로 지어 새 폴더를 만듭니다.
  2. VS Code 메뉴에서 File → Open Folder (단축키 Ctrl + K Ctrl + O) 를 눌러 my-website 를 선택하고 폴더 선택 을 클릭합니다.
  3. 왼쪽 탐색기(Explorer) 에 폴더가 보이면 준비 완료입니다.

STEP 3 Claude Code에게 첫 페이지 만들게 하기
  1. Spark 아이콘 을 눌러 Claude Code 패널을 엽니다.
  2. 아래 문장을 그대로 입력창에 붙여넣고 Enter 를 누릅니다. (이름·내용은 자유롭게 바꿔도 됩니다.)
✓Claude Code에 붙여넣을 문장 (한국어 그대로)
자기소개 웹페이지를 index.html 파일 하나로 만들어줘. 포함: 이름, 사진(플레이스홀더 이미지), 자기소개 3~4줄, 좋아하는 것 3가지(리스트), 이메일. 디자인: 깔끔하고 현대적, 중앙 정렬, 파스텔 톤, 모바일에서도 잘 보이게.
  1. Claude Code가 만들 파일을 Diff(변경 미리보기) 화면으로 보여줍니다. 아래 Accept (체크 표시) 버튼을 누르면 index.html 이 실제로 생성됩니다. (거절은 Reject)
  2. 파일 탐색기(File Explorer) 에서 index.html 을 더블클릭하면 크롬/엣지에서 내 페이지가 뜹니다. 첫 페이지 완성!
  3. 고치고 싶으면 패널에 그냥 말로 부탁하고, 매번 Accept 를 누릅니다.

2) 설치 화면은 모두 기본값 그대로 Next 를 눌러 끝까지 진행 → Install.
3) 설치가 끝나면 VS Code를 완전히 닫았다가 다시 실행하세요(그래야 Git을 인식합니다).
확인: VS Code 아래쪽 터미널(Ctrl + 백틱(`))에 git --version 을 입력해 버전 숫자가 나오면 완료입니다.
  1. 왼쪽 사이드바에서 소스 제어(Source Control) 아이콘을 클릭합니다. (가지 모양, 단축키 Ctrl + Shift + G)
  2. Publish to GitHub 버튼을 클릭하면, 브라우저가 열려 GitHub 로그인 & 권한 허용을 진행합니다(첫 1회만). GitHub 계정이 없으면 github.com/signup 에서 구글 계정으로(Continue with Google) 또는 이메일로 먼저 가입.
  3. 저장소 공개 범위를 고르면(공개/비공개 아무거나) 업로드가 시작됩니다.
  4. 이후 수정할 때마다: 소스 제어 패널의 Message 칸에 간단히 적고(예: "소개 문구 수정") → Commit → Sync Changes (= 올리기). 끝.

STEP 5 Cloudflare Pages로 진짜 사이트 열기
  1. dash.cloudflare.com 에 로그인합니다. 계정이 없으면 가입 화면의 Sign up with Google 로 쓰던 구글 계정으로 바로 가입(가장 쉬움).
  2. 왼쪽 메뉴 Workers & Pages → Create 클릭 → 나오는 화면(Ship something new)을 맨 아래로 스크롤 → 맨 아래 “Pages를 배포하려고 하십니까?” 옆의 시작 링크를 클릭합니다. 위쪽 Continue with GitHub 등은 Workers용이니 누르지 마세요. 예전엔 상단 Pages 탭이었지만 지금은 이 맨 아래 링크로 바뀌었습니다.
  3. 이어지는 “Pages를 시작합니다” 화면에서 기존 Git 리포지토리 가져오기 옆의 시작 버튼을 클릭합니다. (GitHub 없이 파일 끌어서 놓기로 폴더째 올릴 수도 있지만, 여기선 STEP 4에서 올린 GitHub를 씁니다.)
  4. Connect GitHub 클릭 → 뜨는 GitHub 화면에서 Install & Authorize 클릭 → 접근 범위에서 Only select repositories → 방금 올린 my-website 체크(권장) → Cloudflare로 돌아와 my-website 를 선택하고 Begin setup 을 클릭합니다.
  5. 빌드 설정은 정적 HTML이라 그대로 두면 됩니다 — Framework preset = None, Build command = 비움(또는 exit 0), Build output directory = /. → Save and Deploy.
  6. 1~2분 뒤 배포가 끝나면 프로젝트이름.pages.dev 주소가 생깁니다. 클릭하면 전 세계 누구나 볼 수 있는 내 사이트가 열립니다!

빨간 에러 메시지가 나왔어요
그 메시지를 그대로 복사해서 AI에게 붙여넣고 "이 에러가 왜 나는지, 어떻게 고치는지 알려줘"라고 물어보세요. 대부분 그 자리에서 해결됩니다.
화면에 아무것도 안 나와요
"내가 원한 건 A인데, 실제로는 B야. 뭐가 문제인지 찾아줘"처럼 기대한 결과와 실제 결과를 같이 알려주면 AI가 훨씬 잘 찾습니다.
한글이 깨져 보여요
"한글이 깨져. UTF-8로 고쳐줘"라고 하면 됩니다. 이런 사소한 문제일수록 AI가 즉시 해결해 줍니다.
코드가 너무 복잡해 보여요
"이 코드가 어떻게 동작하는지 초보자도 이해하게 설명해줘" 또는 "더 간단하게 만들어줘"라고 부탁하세요. 이해하며 배우는 게 진짜 실력이 됩니다.
