# GitHub 계정 만들기

GitHub는 코드를 저장하고 관리하는 플랫폼입니다. 우리는 GitHub를 통해 Cloudflare Pages와 연결하여 자동 배포를 할 것입니다.

## GitHub가 필요한 이유

- 코드를 안전하게 저장
- 버전 관리 (이전 버전으로 되돌리기 가능)
- Cloudflare Pages와 자동 연동
- 무료로 사용 가능

## 1단계: GitHub 가입하기

### 회원가입
1. https://github.com/signup 접속
2. 이메일 주소 입력
3. 비밀번호 만들기 (최소 15자 또는 8자 + 숫자 + 특수문자)
4. 사용자 이름(Username) 입력
   - 영문 소문자, 숫자, 하이픈(-) 사용 가능
   - 예: `malgeun-kim`, `hopegiver123`
5. 이메일 알림 수신 여부 선택 (y 또는 n)
6. "Continue" 버튼 클릭

### 본인 확인
1. 퍼즐을 풀어 로봇이 아님을 증명
2. "Create account" 버튼 클릭
3. 이메일로 전송된 인증 코드 입력

### 추가 정보 입력 (선택사항)
- 학생/교사인지, 팀 규모 등을 물어볼 수 있습니다
- "Skip" 버튼으로 건너뛰어도 됩니다

## 2단계: 이메일 인증

1. 가입 시 입력한 이메일 확인
2. GitHub에서 온 이메일 열기
3. "Verify email address" 버튼 클릭
4. 브라우저에서 GitHub 로그인 확인

## 3단계: 프로필 설정 (선택사항)

더 전문적으로 보이려면 프로필을 설정해보세요.

1. 우측 상단 프로필 아이콘 클릭
2. "Settings" 선택
3. "Profile" 메뉴에서 설정 가능:
   - Name (실명)
   - Bio (간단한 자기소개)
   - Company (회사명)
   - Location (위치)

## 4단계: GitHub Desktop 설치 (선택사항)

명령어가 어렵다면 GitHub Desktop을 사용하세요.

### Windows / Mac
1. https://desktop.github.com 접속
2. "Download for Windows/Mac" 클릭
3. 설치 파일 실행
4. GitHub 계정으로 로그인

### GitHub Desktop의 장점
- 명령어를 몰라도 사용 가능
- 시각적으로 변경사항 확인
- 클릭 몇 번으로 코드 업로드

## 5단계: 첫 저장소(Repository) 만들기

저장소는 프로젝트를 저장하는 폴더입니다.

### 웹에서 만들기
1. https://github.com 로그인
2. 우측 상단 "+" 버튼 클릭
3. "New repository" 선택
4. Repository name 입력 (예: `my-first-website`)
5. Public 선택 (누구나 볼 수 있음)
6. "Add a README file" 체크
7. "Create repository" 버튼 클릭

축하합니다! 첫 저장소가 만들어졌습니다.

## 용어 설명

### Repository (저장소)
- 프로젝트 하나를 저장하는 공간
- 폴더라고 생각하면 됩니다

### Commit (커밋)
- 변경사항을 저장하는 것
- 사진 찍듯이 그 순간의 코드를 기록

### Push (푸시)
- 내 컴퓨터의 코드를 GitHub에 업로드

### Pull (풀)
- GitHub의 코드를 내 컴퓨터로 다운로드

## 무료 vs 유료 플랜

### Free (무료)
- 무제한 공개 저장소
- 무제한 비공개 저장소
- 웹사이트 호스팅 (GitHub Pages)
- 대부분의 사용자에게 충분

### Pro (유료)
- 고급 도구 추가
- 월 $4
- 초보자에게는 불필요

무료 플랜으로 충분합니다!

## 보안 팁

### 비밀번호 관리
- 강력한 비밀번호 사용
- 다른 사이트와 다른 비밀번호 사용
- 비밀번호 관리자 사용 권장

### 2단계 인증 설정 (권장)
1. Settings > Password and authentication
2. "Enable two-factor authentication" 클릭
3. 인증 앱(Google Authenticator 등) 사용
