# JSON이 뭔가요?

JSON은 데이터를 저장하고 전달하는 가장 간단한 형식입니다.

## JSON이란?

JavaScript Object Notation의 약자. 사람이 읽기 쉽고 컴퓨터가 처리하기 쉬운 데이터 형식입니다.

### 왜 JSON을 사용하나요?

```
데이터베이스 없이 데이터 관리 가능
- 간단한 웹사이트에 적합
- 설정 불필요
- 파일만 만들면 끝

예시:
- 직원 명단
- 제품 목록
- 블로그 글
- FAQ 데이터
```

## JSON의 기본 구조

### 객체 (Object)
중괄호 `{}` 안에 키-값 쌍으로 작성:

```json
{
  "이름": "김철수",
  "나이": 30,
  "직책": "대리"
}
```

### 배열 (Array)
대괄호 `[]` 안에 여러 값을 나열:

```json
[
  "사과",
  "바나나",
  "오렌지"
]
```

### 객체 배열
가장 많이 사용하는 형태:

```json
[
  {
    "이름": "김철수",
    "나이": 30
  },
  {
    "이름": "이영희",
    "나이": 28
  }
]
```

## JSON 데이터 타입

### 문자열 (String)
큰따옴표로 감싸기:
```json
{
  "이름": "홍길동",
  "주소": "서울시 강남구"
}
```

### 숫자 (Number)
따옴표 없이 작성:
```json
{
  "나이": 25,
  "가격": 15000,
  "평점": 4.5
}
```

### 불린 (Boolean)
true 또는 false:
```json
{
  "재직중": true,
  "결혼여부": false
}
```

### null
값이 없음을 표시:
```json
{
  "중간이름": null
}
```

## JSON 작성 규칙

### 1. 키는 반드시 큰따옴표
```json
// 올바른 예
{"이름": "김철수"}

// 틀린 예
{이름: "김철수"}
{'이름': "김철수"}
```

### 2. 마지막 항목 뒤 쉼표 금지
```json
// 올바른 예
{
  "이름": "김철수",
  "나이": 30
}

// 틀린 예
{
  "이름": "김철수",
  "나이": 30,
}
```

### 3. 주석 사용 불가
```json
// JSON에서는 주석을 쓸 수 없습니다
{
  "이름": "김철수"
}
```

## JSON vs CSV

### CSV (쉼표로 구분)
```csv
이름,나이,부서
김철수,30,개발팀
이영희,28,디자인팀
```

### JSON (구조화된 데이터)
```json
[
  {"이름": "김철수", "나이": 30, "부서": "개발팀"},
  {"이름": "이영희", "나이": 28, "부서": "디자인팀"}
]
```

### 언제 JSON을 사용?
- 복잡한 데이터 구조 (중첩된 객체)
- 웹 애플리케이션
- API 통신
- 설정 파일

### 언제 CSV를 사용?
- 단순한 표 형태 데이터
- 엑셀과 호환 필요
- 데이터 분석
